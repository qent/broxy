MCP control plane

One proxy. Sharper tools. Zero drift.

Broxy consolidates MCP servers into a single facade, filters capabilities with presets, and keeps every client aligned with the exact tools, prompts, and resources you want exposed.

Why teams use Broxy

Reduce prompt bloat, enforce routing, and give every agent the exact toolset it needs.

Preset driven filtering

Expose only the tools, prompts, and resources a workflow needs. Keep every agent on scope.

Multi transport proxy

Bridge stdio, streamable HTTP, or WebSocket servers behind one stable endpoint.

ServerId namespace routing

Guaranteed routing with serverId:tool names so calls stay deterministic and auditable.

Fast capability snapshots

Cache capabilities and refresh in the background to keep UI and clients in sync.

Hot reload configs

Update mcp.json and presets without restarting clients. Proxy restarts gracefully.

CLI and desktop UI

Run headless in CI or drive it from the desktop app with the same core runtime.
